## 1 文档概述

### 1.1 编写目的

本操作手册旨在为"汇智海洋——设备检修知识检索与作业系统"的各类用户提供统一、完整的使用说明，帮助一线检修人员、知识管理员及技术专家全面了解系统的功能定位与操作方式，以便在设备检修知识的检索、管理、应用和反馈中高效协作。

手册覆盖用户端和管理端的所有使用场景：帮助一线人员高效检索设备检修知识、通过 AI 智能助手获取故障诊断建议、按标准化模板执行设备检修作业、提交检修案例和客服工单；帮助管理员和专家完成知识库管理、案例审核、用户管理、模型配置及系统运维等工作。

通过提供详细的步骤说明与操作指导，本手册旨在降低系统的学习成本，减少使用错误，提升设备检修知识的管理效率和利用价值，并为后续培训、功能扩展与系统升级提供权威的参考依据。

### 1.2 适用对象

本操作手册适用于"汇智海洋——设备检修知识检索与作业系统"的以下使用人员：

| 对象 | 角色 | 主要使用场景 |
|------|------|------------|
| 一线检修人员 | worker | 浏览和检索知识库、使用 AI 智能问答、执行标准化作业任务、提交检修案例、提交客服工单 |
| 知识管理员 | admin | 管理知识条目和知识库、审核用户提交的案例、管理用户账号、管理系统配置和维护 |
| 技术专家 | expert | 复审经验型检修案例、回复专业客服工单、提供技术指导 |

建议各类用户根据自身角色和权限范围，重点阅读与本角色相关的章节。一线人员请重点阅读第四章"用户端功能操作详细指南"，管理员和专家请同时阅读第四章和第五章"管理端功能操作详细指南"。

### 1.3 软件版本

本操作手册适用于"汇智海洋——设备检修知识检索与作业系统"的初始发布版本（V0.1.0）。该版本为实现全部核心功能的第一版稳定版本。手册将随系统功能的更新和优化进行迭代修订，确保用户能够获得与系统版本相对应的操作指导和注意事项。

### 1.4 编写惯例

| 术语 / 格式 | 说明 |
|------------|------|
| **B/S 架构** | Browser/Server 架构，用户通过浏览器访问系统，无需安装客户端软件 |
| **RAG** | Retrieval-Augmented Generation（检索增强生成），系统先从知识库检索相关内容，再结合大模型生成结构化回答 |

| **【按钮】** | 文中以【】标注界面上的可点击按钮或操作元素 |

## 2 系统简介

### 2.1 系统功能简介

"汇智海洋——设备检修知识检索与作业系统"是基于 B/S 架构的工业设备检修智能化综合辅助平台。系统以多模态大模型技术为核心，将分散的设备手册、维修案例、操作规程、故障经验等知识数字化、结构化，通过 AI 智能对话和语义检索帮助一线检修人员快速获取所需知识，并按标准化作业模板引导检修任务的逐步执行。

**系统核心能力**：

- **知识管理**：支持手动编写、PDF 导入、用户案例上传等多种知识来源。知识内容经过向量化处理后存入 FAISS 向量索引库，支持语义级别的高精度检索。
- **AI 智能助手**：基于 RAG（检索增强生成）技术，结合知识库检索结果和大模型推理能力，提供结构化故障诊断建议。支持 DeepSeek（文本对话）和千问 VL（图文联合理解）两种大模型。
- **作业指引**：管理员预设标准化检修流程模板，一线人员创建任务时系统自动匹配最佳模板，按步骤引导执行，每步附带操作说明和合规校验项。
- **案例上传与审核**：一线人员提交检修经验案例，经过管理员初审和专家复审两级审核后纳入知识库，形成经验沉淀闭环。
- **客服工单**：工单制客服系统，用户遇到 AI 无法解决的问题时可提交工单，管理员/专家在线回复，已解决的工单可沉淀为知识条目。
- **AI 反馈管理**：用户可对每次 AI 回复标注"有用/部分有用/无用"，管理员可查看所有反馈数据，有价值的反馈可转化为知识条目。
- **模型配置**：管理员可动态配置系统使用的大模型 API（支持云端 API 和本地部署），无需重启即生效。

### 4.4 AI 智能助手

#### 4.4.1 功能入口

系统提供两种 AI 智能助手入口：

| 入口 | 访问方式 | 适用场景 |
|------|---------|---------|
| **全屏对话页** | 顶部导航栏 → 【AI 助手】 | 深入的技术咨询和连续的故障排查对话 |
| **悬浮 AI 助手** | 首页或作业执行页右下角的悬浮 AI 按钮 | 快速提问，自动携带当前页面的上下文信息 |

两种入口使用相同的对话引擎（useChat），共享会话历史和消息记录。

#### 4.4.2 全屏对话页概览

全屏对话页采用左侧对话列表 + 右侧对话区域的布局：

| 区域 | 内容 |
|------|------|
| **左侧对话列表** | 【新建对话】按钮、历史对话条目列表（显示标题和更新时间） |
| **右侧对话区域** | 消息显示区（用户消息和 AI 回复气泡）、输入工具栏（输入框、图片上传、模式/模型选择、发送按钮） |

> *（图：AI 全屏对话页整体布局）*

#### 4.4.3 创建新对话

1. 点击左侧面板顶部的【新建对话】按钮
2. 系统自动创建一个空白对话，标题初始为"新对话"
3. 新对话自动切换为当前活跃对话
4. 发送第一条消息后，对话标题自动更新为消息的前 30 个字符

每个对话独立管理，不同对话之间的上下文互不干扰。您可以为不同设备、不同故障创建各自的对话，方便分类管理。

#### 4.4.4 发送文本消息

1. 在底部输入框中输入您的问题（支持多行文本）
2. 按键盘 **Enter** 键发送消息
3. 发送过程中：
   - 输入框不可编辑
   - 发送按钮显示加载状态
   - 消息区底部显示用户消息气泡（蓝色，右侧）
   - AI 回复到达后显示 AI 消息气泡（白色，左侧，带 AI 头像）
4. 发送完毕后输入框恢复可用

> **注意**：不要在输入框中先按 Enter 换行再按 Enter 发送——连续两次 Enter 会发送空消息。如需换行，可使用 Shift+Enter。

#### 4.4.5 选择对话模式

在输入工具栏的【模式选择】下拉框中切换对话模式：

| 模式 | 适用场景 | AI 回复格式 |
|------|---------|-----------|
| **检修模式** (RAG) | 询问设备故障诊断、维修方法、操作规范 | 结构化回答：问题分析 → 可能原因 → 推荐方案 → 参考来源 |
| **闲聊模式** (Casual) | 功能咨询、一般性问答、非检修类对话 | 自然对话风格，不强制结构化，保持对话感 |

检修模式下，系统使用 FAISS 向量索引库进行语义检索获取相关背景知识，再结合大模型生成结构化回复。文本向量化通过 Qwen Embedding API 完成。闲聊模式跳过向量检索，直接以大模型进行对话式回答。

> *（图：对话模式选择下拉框——检修模式/闲聊模式）*

#### 4.4.6 选择模型

在输入工具栏的【模型选择】下拉框中切换使用的大模型：

| 模型 | 能力 | 适用场景 |
|------|------|---------|
| **DeepSeek（文本）** | 纯文本对话 | 常规检修咨询、故障排查问答 |
| **千问 VL（图文）** | 文本 + 图片联合理解 | 上传故障图片进行视觉分析 |

选择千问 VL 后将显示图片上传按钮。模型选择即时生效。

#### 4.4.7 上传图片进行视觉分析

当模型选择为"千问 VL（图文）"时，可使用图片分析功能：

1. 点击输入框旁的图片上传按钮（📷图标）
2. 在文件选择对话框中选择要上传的图片（支持 jpg/png/webp/gif 格式）
3. 已选图片以标签形式显示在上传按钮旁，显示文件名
4. 如选错图片，可点击标签上的关闭按钮移除
5. 在输入框中输入描述性文字（如"请分析图中零件损坏的可能原因"）
6. 点击发送

> **隐私说明**：上传的图片以 base64 编码通过 HTTPS 传输至大模型 API 进行处理。图片不会被保存到系统服务器（仅保存文件名到对话消息记录中）。

#### 4.4.8 切换历史对话

1. 在左侧对话列表中，点击任意历史对话条目
2. 右侧对话区域自动加载该对话的完整消息历史
3. 当前活跃对话以高亮样式标记
4. 可在不同对话间自由切换，消息历史不丢失

#### 4.4.9 重命名对话

1. 在左侧对话列表中，双击需要重命名的对话标题
2. 标题变为可编辑状态
3. 输入新的标题名称
4. 按 Enter 确认，或点击其他区域取消编辑
5. 新标题保存后立即生效

#### 4.4.10 删除对话

1. 在左侧对话列表中，点击对话条目右侧的删除按钮（🗑图标）
2. 系统弹出确认对话框
3. 点击【确定】执行删除
4. 删除后对话及所有关联消息将被永久移除
5. 如果删除的是当前活跃对话，对话区域自动清空

#### 4.4.11 对 AI 回复进行反馈

每条 AI 回复消息下方有三个反馈按钮：

| 反馈 | 按钮 | 操作说明 |
|------|------|---------|
| 👍 有用 | 【有用】 | 直接点击即提交，AI回答准确、对工作有帮助 |
| 🔶 部分有用 | 【部分有用】 | 点击后弹出输入框，可填写改进建议（可选），点击【提交】确认；点击【取消】或关闭弹窗则不提交 |
| 👎 无用 | 【无用】 | 点击后弹出输入框，可填写修正建议（选填，但建议填写以帮助改进），点击【提交】确认；点击【取消】或关闭弹窗则不提交 |

反馈提交后：
- 反馈标记显示在消息下方
- 反馈记录可在个人工作台的"修正记录"标签页中查看
- 管理员可在管理后台的"AI 反馈管理"中查看所有用户的反馈汇总
- **连续两次标记"无用"后**，系统提示"问题未解决？建议提交客服工单获取人工帮助"

> *（图：AI 回复消息气泡下方的反馈按钮——有用 / 部分有用 / 无用）*

#### 4.4.12 悬浮 AI 助手（ChatPanel）

### 4.5 作业指引

#### 4.5.1 任务列表概览

通过顶部导航栏【作业指引】进入。任务列表页面以网格卡片形式展示当前用户的所有检修作业任务。

每张任务卡片包含以下信息：

| 信息项 | 说明 |
|--------|------|
| 任务标题 | 创建时填写的任务名称 |
| 设备型号 | 检修的目标设备型号 |

| 开始日期 | 任务创建日期 |
| 操作按钮 | 进行中/已暂停：点击卡片进入执行页；已完成：显示"查看"按钮 |

页面顶部有【新建任务】按钮。无任务时显示空状态提示。

> *（图：任务列表页面——任务卡片网格布局）*

#### 4.5.2 新建任务

1. 在任务列表页点击【新建任务】按钮
2. 弹出新建任务对话框，包含以下字段：

| 字段 | 说明 | 是否必填 |
|------|------|:---:|
| 任务名称 | 描述本次检修任务，如"Y系列电机季度定修" | ✅ |
| 设备型号 | 从下拉列表中选择或手动输入设备型号。选项来自知识库中已有的设备型号标签 | ✅ |
| 检修等级 | 日常检修 / 定修 / 大修（单选） | ✅ |

3. 选择设备型号和检修等级后，系统**自动触发模板智能匹配**，展示推荐模板列表
4. 从推荐模板中选择一个（点选），或选择"手动输入步骤"
5. 点击【确定】创建任务
6. 任务创建后自动跳转至任务执行页

> **临时模板**：如果选择了手动输入步骤而未选择已有模板，系统会自动创建一个临时模板来存储步骤信息。

#### 4.5.3 模板智能匹配

系统在用户填写设备型号和检修等级后，自动从已发布的模板库中匹配最合适的作业模板。

**匹配评分规则**：

| 匹配项 | 得分 |
|--------|:---:|
| 设备型号精确匹配 | +0.6 |
| 设备型号关键词部分匹配 | +0.3 |
| 检修等级匹配 | +0.3 |

推荐列表按评分从高到低排序，显示匹配百分比，最多展示 Top 5 匹配结果。每项显示模板名称、设备型号、检修等级、步骤数。

#### 4.5.4 查看任务详情

点击任务卡片进入任务执行页。页面布局如下：

| 区域 | 内容 |
|------|------|
| **顶部信息栏** | 任务标题、设备型号、检修等级、进度（当前步骤/总步数） |
| **操作按钮栏** | 【返回】【暂停/恢复】【交接】【提前完成】【AI 问答】 |
| **步骤进度条** | 可视化展示已完成/当前/未完成的步骤 |
| **左侧步骤导航** | 步骤编号 + 阶段 + 标题，当前步骤高亮蓝色，已完成步骤绿色+勾号，未到步骤灰色锁定 |
| **右侧步骤详情** | 当前步骤的详细操作说明和合规校验项 |
| **AI 悬浮按钮** | 右下角，点击弹出 AI 对话面板 |

> *（图：任务执行页面——左侧步骤导航 + 右侧步骤详情）*

#### 4.5.5 执行任务步骤

**进入任务执行页后，按以下流程逐步推进任务**：

1. **阅读操作说明**：右侧步骤详情区展示当前步骤的完整操作说明
2. **勾选合规校验项**：
   - 每个步骤可能包含一个或多个合规校验项（如"执行能量隔离""使用防护手套""确认设备已断电"）
   - 以复选框形式展示，**必须全部勾选才能解锁确认按钮**
   - 未全部勾选时，确认按钮为灰色不可点击状态
3. **确认当前步骤**：全部合规项勾选后，点击【确认并进入下一步】按钮
   - 当前步骤标记为已完成（左侧导航中显示绿色勾号）
   - 自动跳转到下一步
4. **完成所有步骤**：当最后一步确认后，任务状态自动变为"已完成"

> *（图：步骤详情区域——操作说明 + 合规校验项复选框 + 确认按钮）*

#### 4.5.6 暂停与恢复任务

**暂停任务**：

1. 在任务执行页顶部操作栏中点击【暂停】按钮
2. 在弹出的输入框中填写暂停原因（如"等待配件到货""需专家进一步确认"）
3. 点击【确认】后任务状态变为"已暂停"
4. 页面顶部显示暂停提示栏，显示暂停原因

**恢复任务**：

1. 在暂停状态下，顶部操作栏显示【恢复】按钮
2. 点击【恢复】按钮
3. 任务状态恢复为"进行中"，暂停原因自动清除
4. 可继续从当前步骤执行

> *（图：暂停原因输入弹窗）*

#### 4.5.7 交接任务

当任务需要转交给其他班组人员执行时：

1. 在任务执行页顶部操作栏中点击【交接】按钮
2. 弹出交接对话框，包含：
   - **接收人**：从下拉列表中选择同班组或其他人员
   - **交接说明**：填写交接原因和备注（如"已完成拆卸步骤，后续安装由你负责"）
3. 点击【确认】完成交接
4. 交接后：
   - 任务的执行人变更为接收人
   - 原执行人保留为"原始执行人"
   - 交接记录保存到交接链中（记录转出人、接收人、交接时间）

> *（图：交接任务对话框——选择接收人 + 填写交接说明）*

#### 4.5.8 提前完成任务

如需在未执行完所有步骤时提前结束任务：

1. 在任务执行页顶部操作栏中点击【提前完成】按钮
2. 系统弹出确认对话框
3. 点击【确定】确认
4. 系统将当前步骤之后的所有步骤标记为"已跳过"，任务状态变更为"已完成"

该功能适用于中途确认作业已全部完成的场景。

#### 4.5.9 在作业中使用 AI 问答

在任务执行过程中，可随时使用 AI 助手获取当前步骤的智能建议：

1. 在顶部操作栏中点击【AI 问答】按钮（或点击右下角悬浮 AI 按钮）
2. 弹出紧凑对话面板，**自动携带当前设备型号和作业步骤信息作为上下文**
3. 输入问题（如"这个步骤有什么安全注意事项？"），AI 会基于当前步骤和设备型号给出针对性建议

> *（图：任务执行页中的 AI 问答面板）*

#### 4.5.10 删除任务

1. 在任务列表页面，每张任务卡片右下角有【删除】按钮
2. 点击【删除】，系统弹出确认对话框
3. 点击【确定】确认删除
4. 已完成的任务不可删除（需保留归档记录）

### 4.6 案例上传

#### 4.6.1 进入上传页面

通过顶部导航栏【案例上传】进入。案例上传功能让一线人员将检修过程中遇到的案例（包含故障现象、排查过程、根因分析等）提交至系统，经审核通过后纳入知识库供其他人员检索学习。

#### 4.6.2 填写案例信息

1. **案例标题**（必填）：简明扼要描述案例，如"YY型泵机叶轮磨损导致流量下降"
2. **故障描述**（必填）：详细描述故障情况，建议按以下结构组织：
   - 故障现象（观察到什么问题）
   - 初步排查（采取哪些初步检查措施）
   - 深入分析（进一步的技术分析）
   - 根因定位（最终确认的根本原因）
   - 修复措施（采取的具体修复方法）
   - 验证结果（修复后的效果确认）
3. **设备型号**（多选，至少一个）：从下拉列表中搜索选择已有型号，或直接输入新设备型号
4. **故障标签**（多选，至少一个）：从下拉列表选择故障分类标签，或输入新标签
5. **知识类型**（单选）：
   - **事实型**：客观故障现象和修复过程描述；仅需管理员初审
   - **经验型**：包含个人分析判断和调优建议；需管理员初审 + 专家复审（两级审核）

#### 4.6.3 上传现场图片

1. 在"现场图片"区域，点击【上传图片】按钮
2. 选择图片文件（支持 jpg/png/webp/gif，单张最大 10MB）
3. 上传成功后，图片显示在预览列表中
4. 可多次上传，添加多张现场图片
5. 已上传的图片可预览（点击放大查看）和删除
6. 上传的图片文件存储在服务器，返回 URL 路径供后续引用

> **上传流程**：先上传图片获取 URL → 提交案例时在请求中引用图片 URL 列表。这样确保图片已成功落盘后再关联到案例。

#### 4.6.4 提交审核或保存草稿

**提交审核**：

1. 确认案例信息填写完整（标题和故障描述为必填）
2. 点击【提交审核】按钮
3. 系统提示"案例已提交，等待管理员审核"
4. 案例进入审核队列，状态为"待初审"
5. 页面跳转至个人工作台

**保存草稿**：

1. 点击【保存草稿】按钮
2. 案例信息暂存到浏览器本地存储（localStorage）
3. 下次进入案例上传页面时，自动恢复上次填写的草稿内容
4. 草稿仅保存在本地，不会提交至服务器

#### 4.6.5 追踪审核状态

案例提交后，可在以下位置追踪审核进度：

1. 【个人工作台】 → 【我的上传】标签页
2. 案例卡片上显示当前审核状态：
   - 待初审（pending_initial）：等待管理员初审
   - 待复审（pending_expert）：经验型案例初审通过，等待专家复审
   - 已通过（approved）：审核通过，已纳入知识库
   - 已驳回（rejected）：审核驳回，显示驳回原因
3. 已通过的案例可点击链接查看生成的知识条目

4. 清空搜索框或点击清除按钮恢复完整列表

#### 5.3.1 知识条目列表概览

**页面顶部操作栏**：

|----|------|
| 标题 | 知识条目标题（可点击进入编辑页面） |
| 来源 | 手动 / PDF导入 / 用户上传 / 工单转化 / AI反馈转化 |
| 版本 | 当前版本号（如 V1.2） |
| 状态 | 已发布（绿色）/ 草稿（灰色）/ 已归档（橙色） |
| 操作 | 【编辑】按钮 + 【更多 ▾】下拉菜单 |

> *（图：知识库管理列表页面）*

#### 5.3.2 搜索与筛选知识条目

1. 在搜索框中输入关键词，按 Enter 或点击【搜索】按钮
2. 在下拉筛选框中选择目标状态（已发布/草稿/已归档），或选择"全部"
3. 搜索和状态筛选可叠加使用
4. 点击搜索框的清除按钮或清空关键词可恢复完整列表

#### 5.3.3 新增知识条目

1. 在知识列表页点击【新增知识】按钮
2. 进入知识编辑页面，填写以下信息：

| 字段 | 说明 | 必填 |
|------|------|:---:|
| 标题 | 知识条目标题，用于检索和展示 | ✅ |
| 摘要 | 简短描述，列表页展示用 | 选填 |
| 来源类型 | 手动编写 / 工单转化（单选） | ✅ |
| 来源引用 | 溯源信息。手动编写时可填手册名+版本号；工单转化时自动填工单号 | 选填 |
| 设备型号 | 关联的设备型号（多选，支持手动输入新值后按回车添加） | 选填 |
| 故障标签 | 关联的故障分类（多选，支持手动输入新值） | 选填 |
| 检修等级 | 日常检修 / 定修 / 大修（单选） | 选填 |
| 正文内容 | 知识正文，支持 HTML 富文本格式 | ✅ |
| 修改说明 | 编辑时填写，说明本次修改内容（新增时不显示此字段） | 编辑时建议填写 |

3. 如需在正文中插入图片：
   - 点击【上传图片】按钮，选择图片文件
   - 上传成功后图片以 img 标签形式自动插入正文末尾
   - 可在正文中手动调整图片位置
4. 确认信息无误后，点击【创建】按钮保存
5. 新创建的知识条目默认状态为"已发布"

> *（图：新增知识页面——表单字段）*

#### 5.3.4 编辑知识条目

1. 在知识列表中点击目标条目的标题，或点击【编辑】按钮
2. 进入编辑页面，表单预填当前知识条目的所有信息
3. 修改需要更新的内容
4. **在"修改说明"字段中简要描述本次修改的内容**（建议填写，便于版本追溯）
5. 点击【保存】按钮

**版本机制**：每次保存后，系统自动执行以下操作：
- 将当前版本内容保存为历史版本记录
- 版本号自动递增：小版本（V1.0 → V1.1 → V1.2 … → V1.9 → V2.0）
- 生成新的 KnowledgeVersion 记录，包含修改说明和编辑者信息

#### 5.3.5 发布草稿条目

草稿状态的知识条目在用户端不可见（不会被检索和展示），需要发布后才能被用户访问。

1. 在知识列表中，找到状态为"草稿"的条目
2. 点击【更多 ▾】→【发布】
3. 系统弹出确认对话框
4. 确认后，条目状态变更为"已发布"
5. 用户端可立即检索和查看该条目

> PDF 导入和工单/AI 反馈转化的知识条目初始状态均为"草稿"，需管理员确认内容后手动发布。

#### 5.3.6 导入 PDF 手册

1. 在知识列表页点击【导入 PDF】按钮
2. 弹出 PDF 导入对话框：

| 操作项 | 说明 |
|--------|------|
| 【选择文件】 | 点击选取 PDF 文件（支持最大 50MB） |
| 设备型号输入框 | 填写该 PDF 关联的设备型号（选填） |

3. 选择 PDF 文件后，文件名显示在上传区域
4. 点击【开始导入】按钮
5. 系统自动执行以下处理：
   - 提取 PDF 全文文本
   - 提取 PDF 中嵌入的图片
   - 调用千问 VL 模型为每张图片生成中文描述
   - 将文本内容分块，通过 Embedding 模型向量化
   - 将向量存入 FAISS 向量索引库
   - 创建 KnowledgeEntry 记录（状态为"草稿"）
6. 导入完成后，对话框显示导入结果：
   - 知识条目 ID
   - 文本片段总数
   - 向量化片段数
   - 生成图片描述的图片数量
7. 点击结果中的链接"点击编辑此条目 →"跳转到编辑页面
8. 在编辑页面确认内容无误后，点击【发布】使条目在用户端可见

> *（图：PDF 导入对话框——文件选择 + 设备型号输入 + 导入结果显示）*

#### 5.3.7 版本历史与回滚

**查看版本历史**：

1. 在知识列表中，点击目标条目的【更多 ▾】→【版本历史】
2. 进入版本历史页面，表格展示所有历史版本

| 列 | 说明 |
|----|------|
| 版本号 | V1.0, V1.1, V2.0 等 |
| 版本序号 | 整数递增序号（排序用） |
| 修改说明 | 编辑者填写的修改说明 |
| 创建时间 | 该版本的创建时间 |

**回滚到历史版本**：

1. 在版本列表中，找到目标版本行
2. 点击该行的【回滚】按钮
3. 系统将该版本的内容复制为当前知识条目的最新内容
4. 生成一条新版本记录，版本号的大版本 +1（如从 V1.3 回滚到 V1.1 后，生成 V2.0）

> *（图：版本历史页面）*

#### 5.3.8 归档知识条目

1. 在知识列表中，点击目标条目的【更多 ▾】→【归档】
2. 系统弹出确认对话框"确定归档此条目吗？归档后用户端不再可见。"
3. 点击【确定】后条目状态变更为"已归档"
4. 已归档的知识条目在用户端不可检索和浏览，但数据保留在数据库中

#### 5.3.9 删除知识条目

1. 在知识列表中，点击目标条目的【更多 ▾】→【删除】（红色文字）
2. 系统弹出警告确认对话框"确定永久删除此条目吗？所有版本也将被删除，此操作不可撤销。"
3. 点击【确定】后条目及其所有历史版本将被永久删除
4. 删除操作不可恢复，请谨慎操作

> **注意**：删除知识条目不会自动删除 FAISS 中对应的向量数据。如需彻底清理，可通过向量同步接口重新索引。
>
> **向量索引同步**：管理员可访问 `http://服务器地址/api/v1/knowledge/sync-vectors`（需使用管理员账号的 JWT Token 认证）手动触发全文向量重新索引，确保知识库内容变更与 FAISS 索引同步。该接口适用于批量导入或数据恢复后需要重建索引的场景。

#### 5.4.1 审核列表概览

**案例表格**：

|----|------|

#### 5.4.2 按审核状态筛选

1. 点击状态筛选按钮组中的目标状态
2. 表格自动刷新，仅显示该状态的案例
3. 默认显示"待初审"状态

管理员通常关注"待初审"，专家通常关注"待复审"。

#### 5.4.3 查看审核详情

点击案例行进入审核详情页，完整展示案例内容：

| 区域 | 内容 |
|------|------|

| 现场图片 | 用户上传的现场图片列表，点击可放大查看（el-image 的 preview-src-list 功能） |
| 附件列表 | 用户上传的附件文件 |

> *（图：审核详情页——案例内容和图片展示）*

#### 5.4.4 审核操作——通过

用于内容准确、格式完整的案例。

1. 在审核详情页底部操作区，查看案例内容
2. （可选）在"审核意见"输入框中填写评语
3. 点击【通过】按钮
4. 根据案例类型执行不同流程：

| 案例类型 | 执行流程 |
|---------|---------|
| **事实型** | 直接入库：创建知识条目 → 生成版本 V1.0 → 向量化存入 FAISS 向量索引库 → 案例状态变为"已通过" |
| **经验型（初审）** | 状态变为"待复审"，等待专家进行二级审核 |
| **经验型（复审）** | 最终入库：创建知识条目 → 生成版本 → 向量化 → 案例状态变为"已通过" |

5. 提示"审核操作已完成"

> **入库流程说明**：通过后系统自动将案例内容（含图片 HTML）创建为 KnowledgeEntry 记录，来源标记为 user_upload，状态为 published。同时生成初始版本 V1.0，并将内容向量化存入 FAISS 向量索引库。

#### 5.4.5 审核操作——通过（含修改）

用于内容基本可用但需要润色修正的案例。

1. 在审核详情页中，点击【修改内容】折叠按钮
2. 展开内容编辑器，显示案例原始内容
3. 在编辑器中修改和润色案例内容
4. （可选）在"审核意见"输入框中填写评语
5. 点击【通过（含修改）】按钮
6. 系统使用审核员修改后的内容入库（其余流程与"通过"相同）

> *（图：审核详情页——修改内容编辑器展开状态）*

#### 5.4.6 审核操作——驳回

用于内容不完整、信息有误或不符合入库标准的案例。

1. 在审核详情页底部操作区
2. 在"驳回原因"输入框中填写详细的原因说明（必填），如：
   - "故障描述不完整，缺少根因分析"
   - "图片无法清晰展示故障部位，请重新拍摄"
   - "所描述的现象与实际型号不符"
3. 点击【驳回】按钮
4. 案例状态变更为"已驳回"
5. 用户可在个人工作台的"我的上传"中查看驳回原因，修改后重新提交

#### 5.4.7 两级审核机制说明

系统采用两级审核机制，确保知识库质量：

```
                    用户提交案例
                         │
                         ▼
              ┌──────────────────┐
              │   待初审 (管理员)   │
              └────┬──────┬──────┘
                   │      │
          ┌────────┘      └────────┐
          ▼                        ▼
     事实型案例               经验型案例
          │                        │
          ▼                        ▼
    ┌──────────┐           ┌──────────┐
    │ 通过      │           │ 通过      │
    │ → 入库   │           │ → 待复审   │
    └──────────┘           └────┬─────┘
                                │
                                ▼
                      ┌──────────────────┐
                      │   待复审 (专家)    │
                      └────┬──────┬──────┘
                           │      │
                    通过    │      │  驳回
                    → 入库  │      │  → 退回
```

**为什么需要两级审核**：
- **事实型**知识（如"XX零件规格为YY"）只需管理员核查事实准确性和格式完整性
- **经验型**知识（如"根据我的经验，这个故障更可能是XX原因"）包含主观判断，需要技术专家二次验证其技术准确性

### 5.5 用户反馈管理

侧边栏 → 【用户反馈】（可展开子菜单）→ 【客服工单】或【AI 反馈】

#### 5.5.1 客服工单管理——工单列表

管理端工单列表展示所有用户提交的工单，与用户端不同，管理员和专家可看到**所有用户**的工单。

页面顶部提供状态筛选按钮组：

| 筛选项 | 说明 |
|--------|------|
| 全部 | 所有状态的工单 |

| 处理中 | 管理员已接管，正在调查处理 |
| 已回复 | 管理员已给出答复，等待用户确认 |
| 已解决 | 用户确认问题已解决 |

工单表格展示：工单号、标题、提交人、处理人、回复数、状态标签、提交时间。

**操作**：点击工单行进入详情页；每行最右侧有【删除】按钮。

> *（图：管理端工单列表页面）*

#### 5.5.2 客服工单管理——查看与回复

点击工单行进入工单管理详情页。

**页面内容**：

| 区域 | 内容 |
|------|------|
| 顶部信息 | 工单号、标题、当前状态、提交人、提交时间 |
| 问题描述 | 用户的原始问题详细描述 |
| 回复记录 | 完整的回复链，每条回复显示：回复人姓名、回复时间、回复内容 |
| 回复表单 | 文本输入框 + 【发送回复】按钮 |
| 操作按钮 | 【转为知识条目】【关闭工单】【删除工单】 |

**回复工单**：

1. 在底部回复输入框中输入回复内容
2. 点击【发送回复】按钮
3. 首次回复时，工单状态自动从"待处理"变更为"处理中"再变更为"已回复"
4. 回复人自动成为工单的处理人（assignee）
5. 回复记录追加到回复链中

> *（图：工单管理详情页——问题描述 + 回复链 + 回复表单）*

#### 5.5.3 客服工单管理——转为知识条目

将已解决的工单沉淀为知识条目，避免重复回答同类问题。

1. 工单状态必须为"已解决"（RESOLVED）或"已关闭"（CLOSED）
2. 点击【转为知识条目】按钮
3. 系统自动执行：
   - 工单标题 → 知识条目标题
   - 工单问题描述 + 所有回复 → 知识条目正文（Markdown 格式组织）
   - 来源标记为 ticket
   - 状态为 draft（草稿）
   - 作者记为工单提交者
4. 提示"工单已转化为知识条目草稿，请在知识库管理中编辑发布"
5. 管理员可在知识库管理中进一步编辑和发布该条目

#### 5.5.4 客服工单管理——关闭与删除

**关闭工单**：

1. 在工单详情页点击【关闭工单】按钮
2. 工单状态变更为"已关闭"

**删除工单**：

1. 在工单列表页，点击目标工单行的【删除】按钮
2. 系统弹出确认对话框
3. 点击【确定】后工单及其所有回复永久删除

> 用户本人也可以删除自己提交的工单。管理员/专家可删除任意工单。

#### 5.5.5 AI 反馈管理——反馈列表

侧边栏 → 【用户反馈】→【AI 反馈】进入。展示所有用户对 AI 回复的反馈标注。

页面顶部提供反馈类型筛选按钮组：

| 筛选项 | 说明 |
|--------|------|
| 全部 | 所有反馈记录 |
| 无用 | 仅显示标记为"无用"的反馈 |
| 部分有用 | 仅显示标记为"部分有用"的反馈 |
| 有用 | 仅显示标记为"有用"的反馈 |

反馈表格展示：

| 列 | 说明 |
|----|------|
| 用户 | 提交反馈的用户姓名 |
| 用户提问 | 用户当时的问题（截断展示） |
| AI 回复 | AI 的回复内容（截断60字符，省略号标注） |
| 反馈 | 有用（绿色）/ 部分有用（橙色）/ 无用（红色） |
| 修正建议 | 用户提交的修改建议（如有） |
| 时间 | 反馈提交时间 |
| 操作 | 【查看详情】+ 【删除】按钮 |

> *（图：AI 反馈管理列表页面）*

#### 5.5.6 AI 反馈管理——查看详情

1. 在反馈列表中点击目标行的【查看详情】按钮
2. 弹出详情对话框，完整展示：

| 信息 | 说明 |
|------|------|
| 用户 | 提交反馈的用户姓名和 ID |
| 反馈类型 | 有用 / 部分有用 / 无用（标签形式） |
| 用户提问 | 完整的用户原始提问内容（滚动查看） |
| AI 回复 | 完整的 AI 回复内容（滚动查看） |
| 修正建议 | 用户提交的修改建议文本（如无则显示"—"） |
| 时间 | 反馈的具体提交时间 |

3. 对话框底部操作按钮：【关闭】+【转化为知识条目】

> *（图：AI 反馈详情对话框——完整内容展示）*

#### 5.5.7 AI 反馈管理——转为知识条目

将有价值的 AI 反馈转化为知识条目，让用户的问题和修正建议反哺知识库。

1. 在反馈详情对话框中，点击【转化为知识条目】按钮
2. 系统弹出确认对话框
3. 点击【确定】
4. 系统自动执行：
   - 用户提问 → 知识条目标题（截取前 200 字符）
   - 用户提问 + AI 回复 + 用户修正建议 → 知识条目正文（Markdown 格式组织）
   - 来源标记为 ai_feedback
   - 状态为 draft（草稿）
   - 作者记为对话所属用户
5. 提示"AI反馈已转化为知识条目草稿"
6. 自动跳转至知识编辑页面，可进一步编辑后发布

#### 5.5.8 AI 反馈管理——删除反馈

1. 在反馈列表中点击目标行的【删除】按钮
2. 系统弹出确认对话框"确定删除此反馈记录吗？（消息本身保留，仅清除反馈标记）"
3. 点击【确定】后，该消息的 feedback 和 feedback_comment 字段被清空
4. 消息本身保留在对话中不受影响

> **软删除策略**：删除反馈仅清除反馈标记，不影响对话消息的存在。这确保对话历史的完整性。

#### 5.6.2 审核通过新用户

新注册用户的初始状态为"待审核"，无法登录系统。管理员需审核通过后才能激活账号。

1. 在用户列表中找到状态为"待审核"的用户
2. 点击该用户行的【审核通过】按钮
3. 系统向操作日志写入"审核通过用户 XX"

4. 点击【确定】保存
5. 新密码立即生效，用户可使用新密码登录

> 管理员无需知道用户的旧密码即可重置。此功能适用于用户忘记密码的场景。

#### 5.6.5 删除用户

1. 点击目标用户行的【删除】按钮
2. 系统弹出确认对话框
3. 点击【确定】后该用户被永久删除

> **限制**：内置超级管理员账号（admin）不可删除。系统会提示"不能删除超级管理员"。

### 5.9 大模型配置管理

#### 5.9.1 模型配置概览

侧边栏 → 【模型配置】进入。管理员可在此动态配置系统使用的大模型 API，支持用户挂载自己的 API Key，配置变更即时生效无需重启。

**系统自动模型分类**：根据模型名称自动识别功能类型。

| 名称特征 | 功能类型 | 用途 |
|---------|---------|------|
| 含 deepseek / chat / gpt / 文本 | 文本模型 | AI 对话（检修模式/闲聊模式） |
| 含 qwen / vl / vision / 视觉 | 视觉模型 | 图片分析、PDF 图片描述生成 |
| 含 embed / bge / 向量 | 向量模型 | 知识内容向量化、语义检索 |

每种功能类型（文本/视觉/向量）**同时只能有一个激活的配置**。

**配置列表表格**：

| 列 | 说明 |
|----|------|
| 模型名称 | 配置的名称 |
| 模型类型 | 云端 API / 本地部署 |
| API 地址 | 大模型服务的接口 URL |
| API Key | 密钥（仅显示后4位，其余以 * 替代） |
| 激活状态 | 已激活（绿色） / 未激活（灰色） |
| 更新时间 | 最后修改时间 |
| 操作 | 【编辑】【测试】【激活】【删除】按钮 |

> *（图：模型配置列表页面）*

#### 5.9.2 新增模型配置

1. 点击【新增配置】按钮
2. 弹出新增配置对话框，填写：

| 字段 | 说明 | 必填 |
|------|------|:---:|
| 模型名称 | 如 deepseek-chat、qwen-vl-max、text-embedding-v4 | ✅ |
| 模型类型 | 云端 API / 本地部署（单选） | ✅ |
| API 地址 | 大模型服务的接口 URL | ✅ |
| API Key | 用于认证的密钥（密码框输入） | ✅ |
| Temperature | 0-1 滑块，默认 0.3，越低回答越稳定，越高越有创造性 | 选填 |
| Max Tokens | 512-32768 数字输入，默认 8192，控制单次回答最大长度 | 选填 |

3. 点击【保存】
4. 新配置默认未激活，需手动点击【激活】才能生效

> **注意**：新增配置时各字段不预设默认值（如模型名称不自动填充"deepseek-chat"），确保管理员填入的是自己的真实配置。

#### 5.9.3 测试模型连通性

1. 在配置列表中点击目标配置行的【测试】按钮
2. 系统向配置的大模型 API 发送一条简单的测试消息验证连通性
3. 对于向量模型（embedding 类），系统使用 `/embeddings` 端点测试
4. 对于文本和视觉模型，系统使用 `/chat/completions` 端点测试
5. 返回测试结果：成功显示"连接测试成功"，失败显示具体错误信息

> *（图：测试结果提示）*

#### 5.9.4 激活与切换模型

1. 在配置列表中，找到需要激活的配置行
2. 点击【激活】按钮
3. 系统自动将该功能类型下（文本/视觉/向量）的其他已激活配置设为未激活
4. 新配置变为已激活状态，即时生效
5. 下一次 AI 对话或向量检索将使用新激活的模型

**示例**：当前激活的文本模型是 deepseek-chat，如需切换到 qwen-plus：
1. 新增 qwen-plus 配置
2. 点击 qwen-plus 的【激活】
3. deepseek-chat 自动停用，qwen-plus 立即接替

### 6.5 常见问题 FAQ

**Q1：注册后为什么无法登录？**

A：新注册用户的状态为"待审核"（PENDING），需要管理员在管理后台"用户管理"中审核通过后才能登录。如果登录时提示"账号尚未通过审核"，请联系管理员进行审核激活。

---

**Q2：忘记密码怎么办？**

A：系统目前不支持自助找回密码。请联系管理员，管理员可在管理后台"用户管理"中使用"重置密码"功能为您设置新密码。

---

**Q3：为什么 AI 在检修模式下回复较慢？**

A：检修模式下，每次回复需要经过向量检索（从 FAISS 向量索引库搜索相关知识）、Prompt 构建（拼接知识上下文和对话历史）、大模型调用三个环节，通常需要 5-15 秒。如果超过 30 秒无响应：
1. 检查大模型 API 是否正常（管理员可在"模型配置"中测试连通性）
2. 可能是网络波动导致，建议稍后重试
3. 如果频繁超时，可考虑在模型配置中调低 Max Tokens 参数

---

**Q4：如何让 AI 回答更准确？**

A：以下方法可以提高 AI 回答的准确度：
1. 使用**检修模式**（RAG）而非闲聊模式——检修模式会从知识库检索相关知识作为背景
2. 在提问中包含具体的**设备型号**和**故障现象**
3. 上传故障图片让 AI 进行**视觉分析**（需切换到千问 VL 模型）
4. 对不准确的回答点击**"无用"**并填写具体的问题，管理员可通过反馈管理优化知识库

---

**Q5：上传的案例多久能审核通过？**

A：审核时间取决于管理员的工作安排。您可以在【个人工作台】→【我的上传】中随时查看审核状态：
- 待初审：等待管理员处理
- 待复审（仅经验型）：管理员初审已通过，等待专家复审
- 已通过：案例已纳入知识库
- 已驳回：查看驳回原因，修改后可重新提交

---

**Q6：PDF 导入后为什么是草稿状态？**

A：PDF 自动解析的内容可能存在格式问题或文字识别偏差，因此系统将导入的知识条目设为"草稿"状态。管理员需要在知识库管理中：
1. 点击编辑，检查自动解析的内容是否完整准确
2. 必要时手动修正格式和内容
3. 确认无误后点击【发布】

这样确保只有经过人工确认的知识才会被用户检索到。

---

**Q7：为什么不能删除某个设备型号或故障标签？**

A：当设备型号或故障标签被现有知识条目引用时，系统会拒绝删除请求。这是为了防止删除后造成知识条目关联信息丢失。如需删除，请先将所有引用该型号/标签的知识条目修改为其他型号/标签。

---

**Q8：任务执行中能否换人？**

A：可以。在任务执行页面点击【交接任务】，选择接收人员并填写交接说明即可。交接后：
- 原执行人保留为"原始执行人"
- 任务所有权转移给接收人
- 交接记录保存在任务中（含转出人、接收人、时间）

---

**Q9：如何挂载自己的大模型 API Key？**

A：管理员在【模型配置】中操作：
1. 点击【新增配置】
2. 填写模型名称、类型（云端 API / 本地部署）、API 地址和 API Key
3. 保存后点击【激活】
4. 系统自动完成切换，AI 对话和向量检索即时使用新配置

注意：文本/视觉/向量三种模型各有独立的激活槽位，可以同时激活一个文本模型、一个视觉模型和一个向量模型。

---

**Q10：如何备份系统数据？**

A：管理员在【系统设置】中操作：
1. 点击【手动备份】立即执行备份
2. 设置"自动备份周期"为每天/每周/每月，系统按周期自动备份
3. 备份内容包括数据库文件（`app_v2.db`）和上传文件目录（`data/uploads/`）

建议定期备份，并将备份文件存储在服务器之外的独立位置。

---

**Q11：浏览器兼容性如何？**

A：系统推荐使用以下浏览器：
- Google Chrome 90+
- Microsoft Edge 90+
- Mozilla Firefox 88+

不推荐使用 IE 系列浏览器（Internet Explorer）。推荐屏幕分辨率 1440×900 以上以获得最佳显示效果。

---

**Q12：系统出现"数据库已锁定"或长时间无响应怎么办？**

A：此问题通常是由于后台进程异常崩溃导致 SQLite 数据库的日志文件残留。请联系系统管理员：
1. 停止后端服务
2. 检查 `data/app_v2.db-journal` 文件是否存在
3. 删除该 journal 文件（正常关闭时不应存在此文件）
4. 重启后端服务

系统在启动时会自动执行 journal 文件清理，但手动删除可确保 100% 解决此问题。
